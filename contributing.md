\# Contributing to DARSHAN



Thank you for your interest in contributing to \*\*DARSHAN by 8oMATE\*\*.



DARSHAN (Deep Audio-Video Retrieval for Semantic Hybrid Answering Network) is an open-source multimodal AI system designed to analyze video streams using vision, speech, and environmental sound understanding.



We welcome contributions from developers, researchers, and community members who want to help improve the project.



---



\# Code of Conduct



All contributors are expected to follow our \*\*Code of Conduct\*\*.



Please read:



\*\*\[CODE\_OF\_CONDUCT.md](CODE\_OF\_CONDUCT.md)\*\*



before contributing.



---



\# Ways to Contribute



You can contribute to DARSHAN in several ways:



\* Reporting bugs

\* Suggesting new features

\* Improving documentation

\* Fixing bugs

\* Implementing new models

\* Improving system performance

\* Adding datasets or evaluation scripts

\* Enhancing edge-device deployment support



---



\# Getting Started



\## 1. Fork the Repository



Fork the repository to your GitHub account.



Repository link:



https://github.com/nandisagnik/DARSHAN-by-8oMATE



---



\## 2. Clone Your Fork



```bash

git clone https://github.com/<your-username>/DARSHAN-by-8oMATE.git

cd DARSHAN-by-8oMATE

```



---



\## 3. Install Dependencies



```bash

pip install -r requirements.txt

```



If you are contributing to development tools or testing:



```bash

pip install -r requirements-dev.txt

```



---



\## 4. Create a Branch



Always create a new branch before making changes.



```bash

git checkout -b feature/your-feature-name

```



Example branch names:



```

feature/real-time-video-processing

feature/edge-device-support

fix/vector-search-bug

docs/update-readme

```



---



\# Development Guidelines



Please follow these guidelines when contributing.



\### Code Style



\* Follow \*\*PEP 8\*\* Python style guidelines

\* Write clear, readable, and well-documented code

\* Add comments where needed



Recommended tools:



```

black

flake8

```



---



\### Commit Message Style



Use descriptive commit messages.



Examples:



```

feat: add real-time video processing pipeline

fix: resolve chromadb indexing issue

docs: update installation instructions

refactor: improve temporal segmentation module

```



---



\# Submitting a Pull Request



1\. Push your branch to your fork.



```bash

git push origin feature/your-feature-name

```



2\. Open a \*\*Pull Request\*\* on GitHub:



https://github.com/nandisagnik/DARSHAN-by-8oMATE



3\. Provide the following information in your PR:



\* clear description of the change

\* motivation for the change

\* steps to test the change

\* related issue number (if applicable)



---



\# Pull Request Review Process



Maintainers will review pull requests based on:



\* correctness

\* code quality

\* compatibility with project architecture

\* documentation quality



You may be asked to make revisions before your pull request is merged.



---



\# Reporting Bugs



If you encounter a bug, please open a \*\*GitHub Issue\*\* and include:



\* description of the issue

\* steps to reproduce

\* expected behavior

\* screenshots or logs if applicable

\* system information (OS, Python version)



Open an issue here:



https://github.com/nandisagnik/DARSHAN-by-8oMATE/issues



---
